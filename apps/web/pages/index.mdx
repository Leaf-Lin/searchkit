## Features
- **Elasticsearch support** - Searchkit is built on top of Elasticsearch, which means you can use the Elasticsearch query language to build your search experience.
- **Instantsearch support** - Searchkit uses instantsearch, which means you can use the instantsearch libraries to build your search experience.
- **Open source** - Searchkit is open source, which means you can use it for free and contribute to the project.
- **Production Ready** - Searchkit sits in-between Elasticsearch and UI framework, which means it's battle tested and used in production by many companies.

## Relevance Features
- **Query Rules** - Query rules are a powerful feature that allows you to customize the behavior of the search experience. You can use query rules to boost or filter results, or to change the ranking of results. Read more about query rules [here](/docs/guides/query-rules)
- **Full Power of Elasticsearch Query Language** - Searchkit supports the full Elasticsearch query language, which means you can use the Elasticsearch Query DSL to tune your relevance.
- **Synonyms** - Synonyms Support (coming soon)
- **Relevance Tuning** - Relevance Tuning (coming soon)

## Instantsearch Support
With Searchkit and instantsearch, you can build a great search experience with Elasticsearch. Compatible with [Vanilla](https://github.com/algolia/instantsearch.js), [React](https://github.com/algolia/react-instantsearch), [Vue InstantSearch](https://github.com/algolia/vue-instantsearch), [Angular InstantSearch](https://github.com/algolia/angular-instantsearch), and more.

- **Facets** - Facets are a great way to filter your results. Searchkit supports a wide range of facets, including range facets, hierarchical facets, and more.
- **Autocomplete** - Autocomplete is a great way to help users find what they're looking for. Searchkit supports autocomplete for both search and filters.

![overview](../public/searchkit-overview.png)

Searchkit provides two public libraries:

- [API](./api-documentation/api) Node API which transforms instantsearch requests into Elasticsearch queries
- [instantsearch-client](./api-documentation/instantsearch-client) that integrates with the instantsearch library and the Searchkit's SDK.

With Searchkit's API, you'll be able to use instantsearch frameworks to quickly build great search experiences with **Elasticsearch**.

### Setup an API

This creates an API which transforms the instantsearch requests sent from the browser into Elasticsearch queries and transforms the responses into instantsearch results.

<Tabs items={['Next.js', 'Cloudflare Workers', 'express.js']} storageKey="selected-node-platform">
  <Tab>
```ts
import Client from "@searchkit/api";
import { NextApiRequest, NextApiResponse } from "next";

const client = Client({
  connection: {
    host: "<elasticsearch-host>",
    apiKey: "<api-key>", // optional
  },
  search_settings: {
    highlight_attributes: ["title", "actors"],
    search_attributes: ["title", "actors"],
    result_attributes: ["title", "actors"],
    facet_attributes: ["type", "rated"],
  },
});

// example API handler for Next.js
export default async function handler(
  req: NextApiRequest,
  res: NextApiResponse
) {
  const results = await client.handleRequest(req.body);
  res.send(results);
}
```

```ts
import Client from "@searchkit/api";

const client = Client({
  connection: {
    host: "<elasticsearch-host>",
    apiKey: "<elasticsearch api-key>", // optional
  },
  search_settings: {
    highlight_attributes: ["title", "actors"],
    search_attributes: ["title", "actors", "query"],
    result_attributes: ["title", "actors", "query"],
    facet_attributes: [
      "type",
      { attribute: "actors", field: "actors.keyword" },
    ],
  },
});

async function handleRequest(event: FetchEvent) {
  const body = await event.request.json();
  const results = await client.handleRequest(body);

  return new Response(JSON.stringify(results), {
    headers: {
      "content-type": "application/json",
    }
  });
}

addEventListener("fetch", (event) => {
  return event.respondWith(handleRequest(event));
});
```

### Setup the Frontend

Using InstantSearch and the instantsearch-client is as simple as adding this JavaScript code to your page:

<Tabs items={['React', 'Vanilla JS']} storageKey="selected-frontend-platform">
  <Tab>
```tsx
import React from "react";
import ReactDOM from "react-dom";
import Client from "@searchkit/instantsearch-client";
import { InstantSearch, SearchBox, Hits } from "react-instantsearch-dom";

const searchClient = Client({
  url: "/api/search", // API url
});

const App = () => (
  <InstantSearch indexName="bestbuy" searchClient={searchClient}>
    <SearchBox />
    <Hits />
  </InstantSearch>
);

export default App;
```
</Tab>
